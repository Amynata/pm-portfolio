git /cmds

Cheat sheet — commandes essentielles

v 2.x · français
01
git init
Définition

Initialise un nouveau dépôt Git local dans le dossier courant. Crée le répertoire caché .git/ qui contient tout l'historique.

Quand l'utiliser

Au tout début d'un projet, une seule fois, avant tout autre commande Git.

Exemple
$ mkdir mon-projet && cd mon-projet
$ git init
→ Initialized empty Git repository in .git/
02
git add .
Définition

Ajoute tous les fichiers modifiés ou nouveaux à la zone de staging (l'index). Le . signifie "tout le dossier courant".

Quand l'utiliser

Après avoir modifié ou créé des fichiers, juste avant de faire un commit.

Exemple
$ git add .
→ Tous les fichiers sont prêts à être commités
→ Pour un seul fichier : git add index.html
03
git commit -m "message"
Définition

Enregistre un snapshot de la zone de staging dans l'historique. Le -m permet d'écrire le message directement en ligne.

Quand l'utiliser

Après un git add, quand une fonctionnalité ou une correction est terminée. Un commit = une unité logique de travail.

Exemple
$ git commit -m "feat: ajout page contact"
→ [main 3a4f1b2] feat: ajout page contact
04
git push origin <branche>
Définition

Envoie les commits locaux vers le dépôt distant (GitHub, GitLab…). origin est le nom par défaut du remote.

Quand l'utiliser

Après un ou plusieurs commits, pour partager son travail ou sauvegarder en ligne.

Exemple
$ git push origin main
→ Envoie la branche main sur GitHub
$ git push origin feature/login
→ Envoie une branche feature
05
git pull
Définition

Récupère les dernières modifications du dépôt distant et les fusionne automatiquement dans la branche locale. Équivaut à fetch + merge.

Quand l'utiliser

En début de session de travail, ou avant de pusher, pour être à jour avec les changements de l'équipe.

Exemple
$ git pull
→ Récupère et merge la branche courante
$ git pull origin main
→ Récupère explicitement depuis main
06
git checkout <branche>
Définition

Bascule vers une autre branche existante. Avec -b, crée une nouvelle branche et bascule dessus directement.

Quand l'utiliser

Pour changer de contexte de travail : passer sur une feature, corriger un bug sur main, ou créer une nouvelle branche.

Exemple
$ git checkout main
→ Bascule sur la branche main
$ git checkout -b feature/navbar
→ Crée + bascule sur feature/navbar
07
git merge <branche>
Définition

Fusionne l'historique d'une autre branche dans la branche courante. Les commits de la branche source sont intégrés dans la branche de destination.

Quand l'utiliser

Quand une feature est terminée et validée, pour l'intégrer dans main. Toujours se placer sur la branche de destination avant de merger.

Exemple
$ git checkout main ← on se place sur main d'abord
$ git merge feature/navbar ← on intègre la feature dans main
→ Merge made by the 'recursive' strategy.
